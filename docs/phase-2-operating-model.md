# phase 2: operating model

## objective
Prevent multi-agent disorder and increase execution quality.

## components
- Skill graph map (linked markdown nodes)
- Routing rules (minimum viable context)
- Multi-agent protocol (roles + collision control)
- Handoff contracts (structured transfer schema)
- Hook schedule/runbook/state (daily + weekly maintenance)

## role levels
- L1 Observer
- L2 Advisor
- L3 Operator
- L4 Autonomous

## anti-collision rules
- One owner per workstream
- No parallel edits to same target without lock
- Mandatory handoff packet for transfers
